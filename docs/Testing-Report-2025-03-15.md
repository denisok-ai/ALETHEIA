## 2. Найденные ошибки

### 2.1 Консоль браузера

| Тип | Сообщение | Где |
|-----|-----------|-----|
| **Warning** | `Extra attributes from the server: data-cursor-ref` | `PortalSidebar.tsx` (SidebarLogo), `PortalHeader.tsx` (Link). Атрибут `data-cursor-ref` добавляется Cursor IDE для автоматизации и вызывает предупреждение гидратации React |
| **Error** | `No default component was found for a parallel route rendered on this page. Falling back to nearest NotFound boundary` | `/portal/student/courses/[courseId]/play` — страница плеера SCORM |
| **Debug** | `[next-auth][error][CLIENT_FETCH_ERROR] Failed to fetch` | Периодически при навигации (возможно, dev-режим или сетевая задержка) |

### 2.2 Чат-бот

- **Сообщение:** «Сервис ответов временно недоступен. Попробуйте позже.»
- **Причина:** API чата (`/api/chat`) требует настройки LLM (API key и т.п.). Без ключа сервис возвращает fallback-ответ.

### 2.3 Новости — 404 при числовом ID

- **Путь:** `/news/1` → 404
- **Причина:** ID публикаций — строки (nanoid), не числа. Ссылки с главной ведут на корректные ID (из API `/api/publications`).

### 2.4 Плеер SCORM

- Страница `/play` загружается, но в консоли — ошибка parallel route. Визуально контент может отображаться (iframe), но требуется проверка наличия `default.js` для parallel routes, если используется.

## 3. Рекомендации по доработке

### 3.1 Критично

1. **Параллельный маршрут плеера**  
   Добавить `default.tsx` в `app/portal/student/courses/[courseId]/play/` или убрать использование parallel route, чтобы устранить ошибку «No default component was found for a parallel route».

2. **Чат-бот**  
   Настроить переменные окружения для LLM (OPENAI_API_KEY или аналог) в `.env`, чтобы чат отвечал по базе знаний курса.

### 3.2 Важно

3. **data-cursor-ref и гидратация**  
   Атрибут `data-cursor-ref` добавляется инструментами Cursor. Варианты:  
   - Игнорировать в production (атрибут не должен попадать в прод-сборку);  
   - Или исключить его из серверного рендера, если он добавляется на этапе сборки.

4. **Валидация формы заявки**  
   Проверить: пустые поля (Имя*, Телефон*), неверный email — показываются ли toast/сообщения об ошибках.

5. **Кнопки «Сохранить», «Отмена», «Удалить»**  
   Провести точечную проверку в админ-разделах (пользователь, курс, рассылка, группа) на корректность обработки и отображения ошибок API.

### 3.3 Желательно

6. **Документация для новостей**  
   Указать в docs, что ID публикаций — строковые (nanoid), примеры ссылок — из `/api/publications`.

7. **next-auth CLIENT_FETCH_ERROR**  
   При повторяющихся ошибках — проверить NEXTAUTH_URL, CORS и стабильность сессии в dev.

## 5. Дополнительные находки (ручное тестирование)

### 5.1 Модалка покупки курса

- Кнопка «Купить» открывает модалку с выбором тарифа.
- Форма: Email*, Имя*, Телефон (опционально), «Перейти к оплате».
- При нажатии «Перейти к оплате» кнопка меняется на «Создание платежа...» (disabled).
- Без настроенного PayKeeper: после ожидания кнопка возвращается к «Перейти к оплате» — вероятно, API возвращает ошибку, но toast может не отображаться или быть малозаметным.
- **Рекомендация:** при ошибке PayKeeper показывать явный toast с текстом ошибки (например: «Платёжная система недоступна. Обратитесь в поддержку.»).

### 5.2 Конфликт телефонов в футере

- В блоке «Контакты» отображаются два разных номера: +7 (495) 123-45-67 и +7 (999) 123-45-67.
- Источник: `Content.md` / `SystemSetting.contact_phone` vs хардкод в компонентах.
- **Рекомендация:** унифицировать контакты — один источник (настройки).
