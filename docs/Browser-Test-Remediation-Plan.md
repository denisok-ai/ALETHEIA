## Критичные задачи

### 1. 404 на /portal при первом редиректе после логина студента

**Статус:** Исправлено. После signIn вызывается getSession(), по роли определяется дашборд (admin/manager/student), редирект идёт сразу на `/portal/*/dashboard`, минуя `/portal`. Файл: `app/(auth)/login/page.tsx`.

---

### 2. Редирект при signOut на другой порт

**Статус:** Проверено. PortalHeader, PortalAccountBlock, signout используют `callbackUrl: window.location.origin + '/login'`. Редирект остаётся на текущем хосте/порту.

---

## Важные задачи

### 3. SCORM-контент для тестовых курсов

**Статус:** Исправлено. Seed создаёт минимальный SCORM-контент (index.html, lesson1–3.html) для course-seed-1 и course-seed-2 в `public/uploads/scorm/courses-{id}/`. После `npx prisma db seed` плеер показывает контент без загрузки ZIP. Файл: `prisma/seed.ts`.

---

### 4. Клик по карточке курса не всегда переводит на страницу курса

**Статус:** Исправлено. Overlay с play-кнопкой имел `inset-0` и перехватывал все клики. Добавлен `pointer-events-none` на overlay и `pointer-events-auto` на кнопку — клики по обложке проходят к cover div (переход на страницу курса), клики по кнопке — на плеер. Файл: `components/portal/CourseCard.tsx`.

---

## Желательные задачи

### 5. data-cursor-ref и гидратация

**Проблема:** Warning `Extra attributes from the server: data-cursor-ref` в Footer и др. Атрибут добавляется Cursor IDE.

**Действия:** В production атрибута не будет. При необходимости — условный рендер или исключение из серверного HTML. Низкий приоритет.

---

### 6. Hydration-ошибки (localhost:3000)

**Проблема:** `Hydration failed`, `Expected server HTML to contain a matching <header>`. Возникают при редиректе signOut на другой порт (3000).

**Действия:** После исправления п. 2 (callbackUrl) проверить, исчезли ли ошибки. Если остаются — проверить Header/Footer на условный рендер по `typeof window`.
