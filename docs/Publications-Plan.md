# План разработки модуля «Публикации»

Документ составлен на основе мини-ТЗ (инструкция [1–64], макет) и промта для Cursor. Модуль в системе пока отсутствует.

---

## 1. Назначение модуля

Создание и размещение **информационных сообщений (новостей и объявлений)** на страницах системы.

---

## 2. Основные сущности

| Сущность | Описание |
|---------|----------|
| **Публикация** | Основной объект контента (новость или объявление). |

---

## 3. Типы публикаций

| Тип | Поведение |
|-----|------------|
| **Новость** | В списке отображаются заголовок и анонс; полный текст доступен по ссылке. |
| **Объявление** | На странице сразу отображается полный текст сообщения. |

---

## 4. Ключевые поля публикации

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| Название | string | Обязательно | Заголовок публикации. При создании без названия — первые 50 символов контента. |
| Тип | enum | Обязательно | `news` (Новость) / `announcement` (Объявление). |
| Статус | enum | Обязательно | `active` (Действительна, видима) / `closed` (Закрыта, скрыта). |
| Дата размещения | datetime | Обязательно | Дата и время, начиная с которых публикация видна пользователям. |
| Анонс | string | Для новостей | Краткий текст для превью в списках (только для типа «Новость»). |
| Текст | string (HTML) | Обязательно | Полный текст сообщения; поддержка HTML-разметки. |
| Ключевые слова | string[] или string | Опционально | Для поиска и фильтрации (в БД — строка с разделителями или JSON). |
| Просмотры | int | — | Счётчик просмотров (опционально в MVP). |
| Рейтинг | — | — | Средняя оценка и количество (если включено в настройках публикации). |
| Комментарии | — | — | Включение/выключение в настройках публикации; отображение в хронологическом порядке (от старых к новым). |

---

## 5. Логика видимости

- Публикация **видна** пользователям только если:
  - **Статус** = «Действительна» (`active`);
  - **Текущая дата/время** ≥ **Дата размещения**.
- Если статус переведён в «Закрыта» — публикация **мгновенно** исчезает с главной и из каталога, независимо от даты размещения.
- Поле **Анонс** и **Текст** разделены: анонс используется только для превью в списках новостей.

## 6. Модель данных (Prisma)

### 6.1. Публикация (Publication)

```prisma
model Publication {
  id          String   @id @default(cuid())
  title       String
  type        String   // "news" | "announcement"
  status      String   @default("active") // "active" | "closed"
  publishAt   DateTime
  teaser      String?  // анонс (для новостей)
  content     String   // полный текст (HTML)
  keywords    String?  // ключевые слова: строка или JSON-массив
  viewsCount  Int      @default(0)
  ratingSum   Int      @default(0)
  ratingCount Int      @default(0)
  allowComments Boolean @default(true)
  allowRating   Boolean @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  comments    PublicationComment[]
}
```

*(Группы публикаций / рубрики удалены из проекта — модель PublicationGroup не используется.)*

### 6.2. Комментарий (PublicationComment)

```prisma
model PublicationComment {
  id             String   @id @default(cuid())
  publicationId   String
  publication     Publication @relation(fields: [publicationId], references: [id], onDelete: Cascade)
  userId          String?
  user            User?    @relation(fields: [userId], references: [id], onDelete: SetNull)
  authorName      String?   // для гостевых или отображения
  content         String
  createdAt       DateTime @default(now())

  @@index([publicationId])
}
```

- Связь `User` с `PublicationComment`: при необходимости добавить в модель `User` поле `comments PublicationComment[]`.

## 7. Функционал по экранам

### 7.1. Админ-панель

- **Каталог публикаций:** таблица с колонками [№], Название, Тип, Дата размещения, Статус, Просмотры, Действия (Редактировать / Удалить / Предпросмотр).
- **Фильтр и поиск:** фильтр по типу (Все / Новость / Объявление), поиск по названию в реальном времени.
- **Создание/редактирование:** форма с полями:
  - Название (обязательно; подсказка: при пустом — первые 50 символов контента);
  - Тип (переключатель Новость / Объявление);
  - Статус (Действительна / Закрыта);
  - Дата размещения (date-time picker);
  - Анонс (textarea, показывать только если тип = Новость);
  - Текст (редактор с поддержкой HTML);
  - Ключевые слова (строка или теги);
  - Опционально: «Разрешить комментарии», «Разрешить оценку».
- **Удаление** с подтверждением.

### 7.2. Публичная сторона

- **Виджет «Новости» на главной:** блок с последними N (например, 5) **активными** публикациями (статус = active, publishAt ≤ now):
  - Для **новостей:** карточка с названием, датой, анонсом и ссылкой «Читать далее».
  - Для **объявлений:** название, дата и полный текст сразу (или ссылка «Подробнее» по макету).
- **Публичная карточка публикации:** отдельный маршрут (например, `/news/[slug]` или `/publications/[id]`) для чтения полного текста новости/объявления.
- **Социальные функции (MVP):**
  - **Оценка:** компонент 5 звёзд (если у публикации включено «Разрешить оценку»); сохранение в `ratingSum` / `ratingCount`.
  - **Комментарии:** список под контентом в хронологическом порядке (от старых к новым); форма добавления комментария (для авторизованных или гостей — по решению); отображение только если у публикации включено «Разрешить комментарии».

## 8. API

| Метод | Маршрут | Назначение |
|-------|---------|------------|
| GET | `/api/portal/admin/publications` | Список публикаций для админки (с пагинацией/фильтрами при необходимости). |
| POST | `/api/portal/admin/publications` | Создание публикации. |
| GET | `/api/portal/admin/publications/[id]` | Одна публикация (редактирование). |
| PATCH | `/api/portal/admin/publications/[id]` | Обновление публикации. |
| DELETE | `/api/portal/admin/publications/[id]` | Удаление публикации. |
| GET | `/api/publications` или `getPublications()` | Список видимых публикаций для главной и виджета (status=active, publishAt ≤ now). |
| GET | `/api/publications/[id]` или `/[slug]` | Одна публикация для публичной карточки (с проверкой видимости); инкремент просмотра при открытии. |
| POST | `/api/publications/[id]/rate` | Оценка 1–5 (если разрешено). |
| GET | `/api/publications/[id]/comments` | Список комментариев (хронологический порядок). |
| POST | `/api/publications/[id]/comments` | Добавление комментария (авторизованный пользователь или гость — по решению). |

## 11. Критерии приёмки (MVP)

- В админке есть каталог публикаций с созданием, редактированием, удалением и поиском по названию, фильтром по типу.
- На главной странице отображается виджет с последними активными публикациями: для новостей — заголовок, дата, анонс, «Читать далее»; для объявлений — по макету (полный текст или ссылка).
- Есть публичная страница для чтения полного текста публикации.
- Видимость только при статусе «Действительна» и дате размещения ≤ текущего момента; при переводе в «Закрыта» публикация сразу скрывается.
- При создании без названия подставляются первые 50 символов контента (без HTML).
- Опционально: оценка 5 звёзд и комментарии под публикацией (включаются настройками публикации); комментарии в хронологическом порядке.
