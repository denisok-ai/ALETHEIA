# План разработки модуля «Отчётность»

Документ составлен на основе исследования стандартных отчётов в LMS (Moodle, Teachbase, Unicraft, Litmos, LearnWorlds, iSpring, NetDimensions) и специфики мини-школы AVATERRA.

---

## 1. Назначение модуля

**Цель:** предоставить администратору и руководству мини-школы стандартные отчёты в разных разрезах для оценки эффективности обучения, загрузки курсов, активности слушателей и финансовых показателей.

**Аудитория:** администратор портала, владелец школы (при необходимости — менеджер с доступом к отчётам).

---

## 2. Исследование: стандартные отчёты в LMS

### 2.0. Сводка по источникам

**iSpring LMS (16 типов отчётов):** Learner Progress (обзор по всем), Learner Results (по пользователю), Course Details (прогресс по курсу), Assignment Results, Progress Achievements (квизы), Modules report, Group Progress, Enrollment History, Participant Details (посещаемость), Active Users; плюс Answer Breakdown, Leaderboard, Learning Track Progress, Quizzes/Surveys, Certificates (срок действия), Observation Checklists. Экспорт в Excel, планирование рассылки отчётов.

**NetDimensions / Litmos:** Organization (учётные записи, зачисления, транскрипты, компетенции), Learning (оценки курсов, сертификаты, прогресс, использование, модули). Разрезы: по организации, по обучению, по времени.

**Moodle / LearnWorlds:** Learning analytics, логи, отчёты по курсам и студентам. Метрики: вовлечённость, прогресс, завершение, результаты тестов.

**Ключевые разрезы (dimensions):**
- **По пользователям (learners):** прогресс, завершённые курсы, сертификаты, последняя активность, время в системе.
- **По курсам (courses):** зачислено, в процессе, завершило, % завершения, средний балл, время в курсе.
- **По времени (period):** динамика по дням/неделям — зачисления, завершения, выручка, новые пользователи.
- **Сводный (summary):** KPI на один экран — пользователи, курсы, зачисления, завершения, % completion rate, выручка.

**Рекомендации:** фильтрация по периоду/роли/статусу, экспорт CSV/Excel, таблицы с итогами, при необходимости — графики трендов.

### 2.1. Общепринятые категории отчётов

| Категория | Описание | Типичные метрики |
|-----------|----------|------------------|
| **По курсам** | Эффективность и загрузка каждого курса | Зачислено, в процессе, завершило, % завершения, средний балл, время в курсе |
| **По слушателям** | Активность и результаты пользователей | Назначено курсов, завершено, сертификаты, последняя активность, прогресс по курсам |
| **По времени (периоду)** | Динамика за выбранный период | Зачисления по дням/неделям, завершения, выручка, новые пользователи |
| **Сводный (дашборд)** | Ключевые показатели на один экран | Всего пользователей, курсов, зачислений, завершений, % completion rate, выручка |

### 2.2. Рекомендации из практики LMS

- **Экспорт:** возможность выгрузить отчёт в CSV/Excel для анализа вне системы.
- **Фильтрация:** по периоду, по курсу, по роли/статусу пользователя.
- **Визуализация:** таблицы с итогами; при необходимости — простые графики (тренды по дням).
- **Период:** выбор диапазона дат (от — до) или пресеты (7 / 30 / 90 дней, месяц, квартал).

### 2.3. Специфика мини-школы

- Небольшое число курсов и слушателей — отчёты должны быть быстрыми и наглядными.
- Важны: зачисления, завершения, сертификаты, выручка.
- Разрезы: **сводка**, **по курсам**, **по слушателям**, **по периоду**, **детализация по курсу (слушатели курса)**.

### 2.4. Стандартные формы отчётов (реализованные в модуле)

| Форма | Разрез | Назначение |
|-------|--------|------------|
| **Сводка** | Общий | KPI: пользователи, курсы, зачисления, завершения, % завершения, сертификаты, за период — зачисления/завершения/выручка. |
| **По курсам** | По курсу | Таблица: курс, статус, зачислено, доступ открыт, завершило, % завершения, сертификатов, средний балл, время (мин). Фильтр: период, статус курса. |
| **По слушателям** | По пользователю | Таблица: слушатель, email, роль, зачислено, в процессе, завершено, сертификатов, последняя активность, время (мин). Фильтр: период, роль. |
| **По периоду** | По времени | Динамика по дням: дата, зачислений, завершений, сертификатов, оплат, сумма (₽). Итоги за период. |
| **Слушатели курса** | Детализация | По выбранному курсу: слушатель, дата зачисления, доступ, дата завершения, прогресс %, балл, время (мин). |

## 3. Стандартные формы отчётов для AVATERRA

### 3.1. Сводный отчёт (Summary)

**Назначение:** общая картина по школе на дату или за период.

| Показатель | Источник данных | Описание |
|------------|----------------|----------|
| Всего пользователей (активных) | Profile (status=active) | Учётные записи с активным статусом |
| Всего курсов | Course | Все курсы (или только status=published) |
| Зачислений всего | Enrollment | Количество записей |
| Завершивших обучение | Enrollment (completedAt не null) | Количество завершённых участий |
| Сертификатов выдано | Certificate (revokedAt null) | Действующие сертификаты |
| Выручка за период | Order (status=paid, paidAt в периоде) | Сумма amount |
| Зачислений за период | Enrollment (enrolledAt в периоде) | Количество |
| Завершений за период | Enrollment (completedAt в периоде) | Количество |

**Итоговые метрики:** % завершения (завершивших / зачисленных), среднее число курсов на пользователя.

---

### 3.2. Отчёт по курсам (By Course)

**Назначение:** эффективность и загрузка каждого курса.

| Колонка | Описание |
|---------|----------|
| Курс (название) | Course.title |
| Статус курса | Course.status |
| Зачислено | Число записей Enrollment по courseId |
| Доступ открыт | Enrollment где accessClosed = false |
| Завершило | Enrollment где completedAt не null |
| % завершения | (Завершило / Зачислено) × 100 |
| Сертификатов выдано | Certificate по courseId (не отозваны) |
| Средний балл | Среднее по ScormProgress.score по курсу (если есть) |
| Суммарное время (мин) | Сумма ScormProgress.timeSpent по курсу |

**Фильтры:** период (даты зачислений или завершений), статус курса (published / все).

**Экспорт:** таблица в CSV.

---

### 3.3. Отчёт по слушателям (By Learner)

**Назначение:** активность и результаты каждого пользователя (студента).

| Колонка | Описание |
|---------|----------|
| Пользователь (имя / email) | Profile.displayName или User.email |
| Email | User.email / Profile.email |
| Зачислено курсов | Число Enrollment по userId |
| В процессе | Enrollment без completedAt, accessClosed = false |
| Завершено курсов | Enrollment с completedAt |
| Сертификатов | Certificate по userId (не отозваны) |
| Последняя активность | max(ScormProgress.lastUpdated) или последнее enrolledAt |
| Суммарное время (мин) | Сумма ScormProgress.timeSpent по пользователю |

**Фильтры:** период (по дате зачисления или последней активности), роль (user/manager/admin).

**Экспорт:** таблица в CSV.

---

### 3.4. Отчёт по периоду (By Period)

**Назначение:** динамика за выбранный период (по дням или по неделям).

| Разрез | Метрики |
|--------|---------|
| **По дням** | Зачисления (enrolledAt), завершения (completedAt), выданные сертификаты (issuedAt), оплаты (paidAt), сумма оплат |
| **Агрегат за период** | Итого за период: зачислений, завершений, сертификатов, оплат (кол-во и сумма) |

**Формат:** таблица с колонками [Дата | Зачислений | Завершений | Сертификатов | Оплат | Сумма (₽)].

**Фильтры:** дата начала, дата окончания; опционально — группировка по неделям.

**Экспорт:** CSV.

---

### 3.5. Детализация: прогресс по курсу (опционально для MVP)

**Назначение:** по выбранному курсу — список слушателей с прогрессом.

| Колонка | Описание |
|---------|----------|
| Слушатель | displayName / email |
| Зачислен | enrolledAt |
| Доступ | открыт / закрыт |
| Завершён | completedAt |
| Прогресс % | доля завершённых уроков (ScormProgress completionStatus=completed) от общего числа уроков курса |
| Балл | средний/итоговый score по курсу |
| Время (мин) | сумма timeSpent по курсу |

Реализуется как отдельный отчёт или расширение «По курсам» (переход к списку слушателей курса).

## 4. Техническая реализация

### 4.1. API

| Метод | Маршрут | Назначение |
|-------|---------|------------|
| GET | `/api/portal/admin/reports/summary` | Сводный отчёт (query: dateFrom, dateTo — опционально период) |
| GET | `/api/portal/admin/reports/by-course` | Отчёт по курсам (query: dateFrom, dateTo, status) |
| GET | `/api/portal/admin/reports/by-learner` | Отчёт по слушателям (query: dateFrom, dateTo, role) |
| GET | `/api/portal/admin/reports/by-period` | Отчёт по периоду — динамика по дням (query: dateFrom, dateTo) |
| GET | `/api/portal/admin/reports/course/[courseId]/learners` | Детализация: слушатели курса с прогрессом (прогресс %, балл, время, сертификат) |

Формат ответа: JSON. Для экспорта — тот же JSON, на клиенте формируется CSV (или отдельный query-параметр `?format=csv` и Content-Type: text/csv).

### 4.2. Админ-интерфейс

- **Раздел «Отчётность»** в меню админки.
- **Выбор типа отчёта:** карточки или вкладки: Сводка | По курсам | По слушателям | По периоду.
- **Параметры:** период (dateFrom, dateTo), при необходимости — фильтр по статусу курса / роли.
- **Отображение:** таблица с данными; кнопка «Экспорт в CSV».
- **Сводка:** можно показать в виде карточек с ключевыми цифрами + небольшая таблица (как на дашборде, но с акцентом на отчётность).

### 4.3. Безопасность

- Все маршруты отчётов — только для роли **admin** (requireAdminSession).
- Экспорт — те же данные, что и на экране, без расширения прав.
