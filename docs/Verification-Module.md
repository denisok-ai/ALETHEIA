# Модуль «Верификация заданий» (Phygital)

**Назначение:** проверка практических заданий курса: студент записывает видео (или прикладывает ссылку), отправляет на проверку; менеджер/админ одобряет или отклоняет. Подходит для курсов школы AVATERRA, где часть заданий — «физические» (показать технику, выполнить упражнение), а отчёт — «цифровой» (видео).

---

## 1. Что это за модуль

| Аспект | Описание |
|--------|----------|
| **Название** | Phygital = physical + digital: практика в реальности, отчёт в виде видео/ссылки. |
| **Роли** | **Студент** — выполняет задание, прикрепляет видео (ссылку или загрузку). **Менеджер/админ** — смотрит очередь, одобряет или отклоняет с комментарием. |
| **Связь с курсом** | Задание привязано к курсу и опционально к уроку (lessonId из SCORM). Одобрение может влиять на прогресс/сертификат (если доработать логику). |

**Модель данных (уже есть):** `PhygitalVerification`: userId, courseId, lessonId?, videoUrl, status (pending | approved | rejected), reviewedBy?, reviewedAt?, comment?, createdAt.

---

## 2. Как сейчас работает (реализовано)

- **Менеджер:** раздел **Портал → Верификация заданий** (`/portal/manager/verifications`). Отображается очередь записей со статусом `pending`: ФИО студента, курс, урок, ссылка «Смотреть видео», кнопки «Одобрить» / «Отклонить». После действия запись исчезает из списка (статус меняется на approved/rejected). В дашборде менеджера выводится счётчик «На верификации».
- **Данные:** записи создаются в seed для теста; в проде они должны появляться из заявок студентов.

**Реализовано дополнительно (продолжение разработки):**

- **Студент:** на странице курса (`/portal/student/courses/[courseId]`) — блок «Задания на проверку»: выбор урока (из SCORM-manifest), поле «Ссылка на видео», кнопка «Отправить на проверку»; краткая инструкция в блоке. API `POST /api/portal/verifications` (courseId, lessonId?, videoUrl). Страница «Мои отправки» (`/portal/student/verifications`) — список отправок со статусом (на проверке / одобрено / отклонено), ссылка на видео, комментарий менеджера при отклонении. В меню студента добавлен пункт «Задания на проверку».
- **Помощь:** в разделе «Помощь» добавлен блок «Как отправить задание на проверку» (id=verification для якоря).
- **Менеджер:** при отклонении можно указать комментарий для студента; комментарий сохраняется в БД и отображается в «Моих отправках» у студента.
- **Менеджер (страница верификации):** отображаются полное название курса, ссылка на карточку пользователя (для роли admin), адаптивная сетка на всю ширину.
- **Студент (страница курса):** в блоке «Задания на проверку» — список «Отправки по этому курсу» (статус по каждой отправке: на проверке / одобрено / отклонено, комментарий при отклонении).
- **Админка курса:** блок «Уроки с верификацией» — выбор уроков из SCORM-манифеста, по которым студент должен отправить видео; сохраняется в `Course.verificationRequiredLessonIds` (JSON array). На странице курса у студента выводится подсказка «По этому курсу нужно отправить видео по урокам: …».
- **Загрузка видео:** на странице «Задания на проверку» и в блоке на странице курса — кнопка «Загрузить видео» (файлы MP4, WebM, MOV, AVI, MKV до 200 МБ). API `POST /api/portal/verifications/upload` сохраняет в `public/uploads/verifications/`, возвращает URL; при редактировании задания (pending) тоже доступна загрузка.
- **Связь с сертификатом:** при наличии у курса `verificationRequiredLessonIds` авто-выдача и массовая выдача сертификатов требуют одобренной верификации по каждому из этих уроков.

## 3. Где должно быть прикрепление заданий со стороны студента

Возможные варианты:

| Вариант | Где | Плюсы | Минусы |
|---------|-----|--------|--------|
| **A. В карточке курса** | На странице курса студента (`/portal/student/courses/[courseId]`) — блок «Задания на проверку»: список уроков/заданий с кнопкой «Отправить видео» (ссылка или загрузка). | Контекст курса и урока сразу ясен. | Нужно знать, какие уроки требуют верификации (метаданные курса/SCORM). |
| **B. Отдельная страница** | Раздел **«Задания на проверку»** в сайдбаре студента (`/portal/student/verifications`): список курсов и заданий, форма «Новая отправка» (курс, урок, ссылка/файл). | Один вход для всех заданий. | Менее привязано к прохождению урока. |
| **C. Внутри SCORM** | В контенте урока (в самом пакете) — кнопка «Отправить видео на проверку» с вызовом API или переходом на портал с подставленными courseId/lessonId. | Максимально в контексте урока. | Зависит от возможностей SCORM и доработки пакетов. |

**Рекомендация:** начать с **A** (блок на странице курса) + при необходимости **B** (отдельная страница «Мои отправки» для истории). Так студент видит задание в контексте курса и урока; при желании можно дублировать ссылку «Задания на проверку» в сайдбаре, ведущую на список своих отправок по всем курсам.

## 4. Кто и где подсказывает, как записать видео и прикрепить

Сейчас ни один элемент интерфейса этого не делает. Имеет смысл добавить подсказки в нескольких местах.

| Место | Содержание |
|-------|------------|
| **Страница курса (блок заданий)** | Краткая инструкция: «Для заданий с проверкой: запишите короткое видео (телефон/камера), загрузите на YouTube / облако или загрузите файл здесь, затем прикрепите ссылку или выберите файл и нажмите «Отправить на проверку». Менеджер проверит в течение 1–2 рабочих дней.» При необходимости — ссылка «Подробная инструкция» на раздел «Помощь». |
| **Раздел «Помощь» (студент)** | Отдельный подпункт **«Как отправить задание на проверку»**: что такое верификация, как записать видео (формат, длительность, что должно быть в кадре — по методике школы), куда загрузить (YouTube unlisted, Google Drive с доступом по ссылке, или загрузка в портал), как вставить ссылку или прикрепить файл в ЛК, когда ждать ответа. |
| **Описание курса / урока** | В админке у курса или в SCORM-контенте урока — текст задания и напоминание: «Отчёт — видео; отправить можно на странице курса в блоке «Задания на проверку».» Тогда студент видит это при прохождении курса. |

Ответ на вопрос «кто подсказывает»: **сам портал** — через блок заданий на странице курса и раздел «Помощь». При необходимости методист/админ дополняет тексты в описании курса и в контенте уроков (админка / редакция SCORM).

## 5. План доработок (кратко)

1. **Студент — отправка задания**
   - На странице курса (`/portal/student/courses/[courseId]`) добавить блок «Задания на проверку»: какие уроки/задания требуют видео, форма «Ссылка на видео» или «Загрузить файл» (если реализуем загрузку), кнопка «Отправить на проверку». API: например `POST /api/portal/student/verifications` (courseId, lessonId?, videoUrl или file).
   - Опционально: страница `/portal/student/verifications` — «Мои отправки» (история по всем курсам, статусы pending/approved/rejected, комментарий менеджера при отклонении).

2. **Инструкции для студента**
   - В блоке заданий на странице курса — короткий текст (как записать, куда залить, что нажать).
   - В **Помощь** (`/portal/student/help`) — раздел «Как отправить задание на проверку» с пошаговой инструкцией и, при необходимости, рекомендациями по съёмке (по согласованию с методистом).

3. **Менеджер (уже есть)**
   - При желании: при отклонении — поле «Комментарий» студенту и отображение этого комментария в «Моих отправках» у студента. В модели уже есть поле `comment`.

4. **Связь с прогрессом/сертификатом (опционально)**
   - Правило вида: «урок считается пройденным для сертификата только после одобренной верификации» — потребует доработки логики подсчёта completed-уроков и, возможно, флага в настройках курса/урока «требует верификации».
