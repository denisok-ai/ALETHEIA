# Результаты обзорного теста в браузере (4 апреля 2026)

**URL:** http://localhost:3000  
**Инструмент:** встроенный браузер Cursor (автоматизация кликов/навигации).  
**Ограничения сессии:** не выполнялось полное покрытие всех разделов админки/менеджера/студента, не создавались и не удалялись тестовые сущности через UI (риск для вашей БД, большой объём). Рекомендуется дополнить прогоном `npm run test:e2e` и ручным чек-листом.

---

## Критичные находки

| # | Наблюдение | Влияние |
|---|------------|---------|
| 1 | **Главная `/` отдаёт HTTP 500** (страница ошибки браузера) для неавторизованного доступа | Гость не видит лендинг; ссылки «Главная» из футера ведут на падение; SEO и первый контакт с сайтом сломаны |
| 2 | В консоли при сценарии входа: **`Failed to fetch RSC payload`**, **`[next-auth][error][CLIENT_FETCH_ERROR] Failed to fetch`** | Нестабильная навигация/сессия в dev; на проде проверить `NEXTAUTH_URL`, сеть, прокси |

**Рекомендация по п. 1:** в терминале dev-сервера посмотреть stack trace при запросе `GET /`; типичные причины: падение `getSystemSettings()`, БД, импорт секции лендинга, внешний API при SSR.

---

## Проверенные сценарии (успешно)

- **Выход:** `/api/auth/signout` → страница подтверждения «Выход» → после подтверждения редирект на `/login`.
- **Вход:** форма логина (email/пароль), редирект в портал студента (`student4@test.local`).
- **Студент:** дашборд `/portal/student/dashboard` — курсы, заряд, уведомления, подсказка (`role="status"`), навигация.
- **RBAC:** под студентом переход на `/portal/admin/dashboard` → **`/portal/access-denied?section=admin`** с понятным текстом и ссылкой «Личный кабинет».
- **Менеджер** (другая вкладка/сессия): дашборд менеджера, счётчики тикетов/верификаций, ссылки на тикеты; ранее в сессии — **access-denied** для админки с «Кабинет менеджера».

## Замечания по UI / доступности

1. **Дублирование контролов на дашборде студента:** в дереве доступности одновременно **ссылка** и **кнопка** с подписью «Мои курсы» (и аналогично «Поддержка») — возможна путаница для клавиатуры/скринридеров; стоит оставить один интерактивный паттерн или чётко развести роли (`aria-hidden` на дубликате — только если семантика продумана).
2. **Заголовок `Студент 4 👋`:** эмодзи в `<h1>` может некрасиво озвучиваться в NVDA/VoiceOver; вынести в отдельный декоративный span с `aria-hidden` или убрать из имени заголовка.
3. **Футер:** в снимке доступности встречается фрагмент **«© АВАТЕРРА . ИП…»** — лишний пробел перед точкой (мелочь копирайта).
4. ~~**Копирайт в layout авторизации**~~ — заменено на «← На сайт».

## Что не было проверено в этой сессии (продолжить вручную / E2E)

- Все разделы **администратора** (курсы, пользователи, настройки, отчёты, почта, AI, медиа и т.д.).
- Полный проход **менеджера** (таблицы тикетов/верификаций с фильтрами, массовые действия, Excel).
- **Гость:** лендинг (заблокирован 500), контакты, FAQ, модалка покупки, регистрация.
- Сквозное **создание объекта** (тикет, публикация, курс, товар…) со всеми полями, проверка под другими ролями и **удаление** — выполнить по чек-листу команды на стенде-копии БД.

## После отчёта (сделано в коде)

- **`lib/settings.ts`:** при ошибке БД `getSystemSettings` не роняет страницу — fallback из `ENV` / пустые значения, лог в консоль сервера.
- **Дублирование «АВАТЕРРА» в шапке/футере/сайдбаре:** у `BrandLogo` флаг `withVisibleBrandText` — не дублировать `sr-only`, если рядом уже есть название или `aria-label` у ссылки.
- **Подсказка студенту:** ссылки «Мои курсы» / «Поддержка» — один элемент (`<Link>` со стилями кнопки), без вложенной `<button>`.
- **Приветствие:** эмодзи вынесено в `aria-hidden`.
- **Auth layout:** «← На сайт» вместо «← Аватера».
- **Футер:** неразрывный пробел после «©».
