# Аудит: куда можно добавить AI-помощников

Обзор уже используемого AI и мест, куда имеет смысл встроить генерацию/подсказки. Основа — текущий код и структура портала.

---

## 1. Уже реализовано

| Место | Назначение | Ключ/API |
|-------|------------|----------|
| **Чат-бот на сайте** | Публичный чат на главной (ChatBot.tsx → /api/chat) | LlmSetting `chatbot` |
| **AI-тьютор в курсе** | Чат в SCORM-плеере по контенту курса (CourseAIChat → /api/portal/scorm/ai-assist) | LlmSetting `course-tutor` или `chatbot` |
| **Генерация промптов** | Кнопка «Сгенерировать» в шаблонах промптов (PromptTemplatesBlock) | `chatbot` |
| **Генерация обложки курса** | Кнопка «AI Сгенерировать обложку» в карточке курса (CourseCoverBlock) | OPENAI_API_KEY (DALL-E 3) |
| **Настройки AI** | Страница /portal/admin/ai-settings: ключи, модели, шаблоны, база знаний, тест чата | — |
| **Тикеты поддержки** | Кнопка «AI предложить ответ» в форме ответа (TicketThread), для менеджера/админа | POST …/tickets/[id]/suggest-reply, ключ `chatbot` |
| **Шаблоны коммуникаций** | Кнопка «AI сгенерировать» в форме шаблона (тема + тело по названию и каналу) | POST …/ai-settings/generate-text |
| **Шаблоны уведомлений** | Кнопка «AI сгенерировать» в NotificationTemplateForm (тема + текст) | POST …/ai-settings/generate-text |
| **Рассылки** | Кнопка «AI предложить тему и текст» в форме рассылки | POST …/ai-settings/generate-text |

## 3. Технические общие моменты

- **Единый API генерации текста**  
  Имеет смысл вынести общий endpoint, например `POST /api/portal/admin/ai-settings/generate-text`, с телом `{ instruction: string, context?: string, maxTokens?: number }`, и использовать его для тикетов, шаблонов, рассылок, курсов, публикаций, CRM, медиа. Ключ — `chatbot` или отдельный ключ для админских подсказок.

- **Ключи**  
  Текстовые подсказки: существующие LlmSetting (`chatbot` или новый `content-assistant`). Изображения: по необходимости оставить OPENAI_API_KEY для DALL-E (обложки и т.п.).

- **Ограничения**  
  Генерация только как черновик/подсказка: пользователь всегда может править результат. Не подставлять ответ в тикет и не отправлять письма без явного действия пользователя.

- **Логирование**  
  Реализовано: in-memory лог последних 100 запросов к LLM (lib/llm-request-log.ts). На странице Настройки AI блок «Лог запросов к AI» показывает таблицу (дата, сценарий, модель, символы промпт/ответ, время мс, роль). Логируются: чат-бот (POST /api/chat), подсказка ответа в тикете (suggest-reply), автоответ при создании тикета (ticket-auto-reply), генерация текста (generate-text), генерация промпта (prompt-generate). Лог сбрасывается при перезапуске сервера. GET /api/portal/admin/ai-settings/request-log (только админ).

- **Уже есть:** чат-бот, AI-тьютор в курсе, генерация промптов, генерация обложки курса, настройки AI.
