## 1. Критические ошибки UI

### 1.1 Уведомления на дашборде студента — сырой JSON вместо текста ✅

**Где:** `/portal/student/dashboard`, блок «Уведомления»

**Проблема:** Вместо читаемого текста отображается JSON: `{"title":"Уведомление 4","ref":"ref-3"}` или `{"subject":"...","body":"..."}`.

**Причина:** Поле `Notification.content` хранит JSON (subject/body от triggerNotification или title/ref в seed). Компонент выводит `String(n.content)` без парсинга.

**Решение (выполнено):**
- Создана утилита `lib/notification-content.ts` с функцией `formatNotificationContent(content, type?)`.
- В `app/portal/student/dashboard/page.tsx`: используется `formatNotificationContent(n.content, n.type)`.
- В `app/portal/student/notifications/NotificationsList.tsx`: используется та же утилита.
- Счётчик уведомлений в PortalHeader выводит только число (точку), не content — правок не требуется.

**Приоритет:** Высокий

### 1.2 Перекрытие кнопки «Отправить заявку» на главной ✅

**Где:** Главная страница, форма контакта (секция «Оставьте заявку»)

**Проблема:** Кнопка «Купить курс» (sticky CTA или блок магазина) перекрывает кнопку «Отправить заявку» — клик перехватывается другим элементом. Пользователю приходится прокручивать страницу, чтобы отправить форму.

**Решение (выполнено):**
- В `StickyCTA.tsx`: sticky CTA скрывается, когда секция #contact в зоне видимости (аналогично footer).
- В `Contact.tsx`: добавлены `scroll-mt-20` и `scroll-mb-[100px]` для якоря #contact — при переходе по ссылке форма видна и не перекрыта.

**Приоритет:** Высокий

### 1.3 SVG path: invalid d attribute (иконка валюты) ✅

**Где:** `/portal/admin/dashboard`, KPI-блок «Выручка (мес.)»

**Проблема:** В консоли: `Error: <path> attribute d: Expected number, "… 2.032 0 00-.892.hands zM10 …"`. В SVG path была опечатка `.hands` вместо `.5`.

**Решение (выполнено):** В `app/portal/admin/dashboard/page.tsx` исправлен path: `.892.hands` → `.892.5`.

**Приоритет:** Средний

## 2. Ошибки Recharts (дашборд админа)

### 2.1 width(-1) height(-1) у графиков ✅

**Где:** `/portal/admin/dashboard`, блоки «Выручка по дням» и «Активность по дням»

**Проблема:** В консоли: `The width(-1) and height(-1) of chart should be greater than 0...`

**Решение (выполнено):**
- Родитель: `min-h-[256px]`, `style={{ minWidth: 0 }}` для flex-контейнера.
- ResponsiveContainer: `minWidth={0}` и `minHeight={100}`.
- Файл: `app/portal/admin/dashboard/DashboardCharts.tsx`.

**Приоритет:** Средний

## 3. Ошибки контента и орфографии

### 3.1 Опечатка «проходеним» → «прохождением» ✅

**Где:** Дашборд менеджера, последние тикеты — «Проблема с проходеним курса»

**Решение (выполнено):** В `prisma/seed.ts` добавлена тема «Проблема с прохождением курса» (правильное написание) в массив `ticketSubjects`. Новые seed-данные будут с корректной орфографией. Существующие записи в БД — пользовательский ввод, не меняются.

**Приоритет:** Низкий

---

### 3.2 Склонение «сертификат(ов)» ✅

**Где:** Дашборд студента — «5 сертификат(ов)»

**Проблема:** Для 2, 3, 4 должно быть «сертификата», для 5+ — «сертификатов». Текущая форма «сертификат(ов)» — компромисс, но лучше использовать правильное склонение.

**Решение (выполнено):** Создана `lib/pluralize.ts` с функцией `pluralize(n, one, few, many)`. На дашборде студента используется `pluralize(certCount, 'сертификат', 'сертификата', 'сертификатов')`.

**Приоритет:** Низкий

## 4. Потенциальные проблемы (требуют проверки)

### 4.1 Одинаковые темы тикетов у студента

**Где:** `/portal/student/support`, список «Мои обращения»

**Наблюдение:** В тесте у студента 4 все тикеты отображались как «Технический вопрос — тикет 4». Seed создаёт разные темы по `subjectIdx = i % 6`. Возможные причины: особенности seed (много тикетов с одной темой для одного студента) или баг отображения. Требуется проверка на свежей БД после `npx prisma db seed`.

**Приоритет:** Средний

---

### 4.2 next-auth CLIENT_FETCH_ERROR

**Где:** Консоль браузера при загрузке `/portal/student/dashboard` и других страниц портала

**Сообщение:** `[next-auth][error][CLIENT_FETCH_ERROR] Failed to fetch [object Object]`

**Примечание:** Может быть временной ошибкой сети или особенностью dev-окружения. Рекомендуется проверить в проде и при стабильном воспроизведении — логировать и обрабатывать.

**Приоритет:** Низкий

## 5. Функциональные проблемы (SCORM, progress, формы)

### 5.1 SCORM progress — обновление на карточке курса

**Где:** Дашборд студента, карточки курсов; SCORM-плеер

**Проверка:** После коммита прогресса в SCORM (autocommit каждые 15 с) карточка курса на дашборде должна обновить балл/время. Дашборд — SSR, карточки — статичные при загрузке. Обновление возможно только при навигации или refresh.

**Рекомендация:** Добавить периодический refetch или SWR для карточек курсов на дашборде студента, если нужна «живая» статистика без перезагрузки.

**Приоритет:** Низкий
