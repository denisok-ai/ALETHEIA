# Аудит: обращения поддержки (тикеты), интерфейсы и AI

## 1. Приходят ли обращения в интерфейс менеджеров и администраторов?

| Роль | Где видит обращения | Ответ |
|------|---------------------|--------|
| **Менеджер** | Раздел **«Тикеты»** (`/portal/manager/tickets`) — список всех тикетов. Дашборд — счётчик открытых тикетов и последние 5. Письмо на `resend_notify_email` при новом обращении. | **Да** |
| **Админ** | Дашборд — счётчик открытых тикетов и ссылка на тикеты. В карточке пользователя (вкладка «Тикеты») — список тикетов пользователя со ссылкой «Открыть в портале» → `/portal/manager/tickets`. Отдельного раздела «Тикеты» в сайдбаре админки **не было** — добавлена ссылка «Тикеты» в блок «Коммуникации», ведущая на тот же список (`/portal/manager/tickets`). | **Да** (через общий с менеджером интерфейс) |

Итог: обращения создаются студентом в «Поддержка» → «Ваши обращения и заявки», сохраняются в БД, приходят на email менеджеру и отображаются в разделе «Тикеты» (доступен и менеджеру, и админу).

---

## 2. Есть ли интерфейс для ответа на вопросы?

**Да.** Менеджер/админ открывает тикет по ссылке из списка или из карточки пользователя:

- **Страница тикета** `/portal/manager/tickets/[id]`: тема, автор, статус, назначенный менеджер, переписка, поле «Ответ», кнопка «Отправить».
- Дополнительно: смена статуса (Открыт / В работе / Решён / Закрыт), назначение менеджера, выбор **шаблона ответа** (несколько готовых текстов), кнопка **«AI предложить ответ»** — подстановка сгенерированного текста в поле ответа (редактируемый, отправка вручную).

Ответ сохраняется в тикете и уходит студенту по email (если настроена отправка уведомлений о новых сообщениях в тикете — при наличии такой логики).

---

## 3. Есть ли автоответы в режиме AI?

**Сейчас: нет.** При создании обращения студентом:

- Студенту уходит только шаблонное письмо «Обращение принято».
- Менеджеру — уведомление о новом тикете.
- Автоматического первого ответа от AI при создании тикета нет.

**Рекомендация:** ввести опциональный автоответ AI при создании тикета (настройка в админке «Настройки AI» или «Коммуникации»): по теме/тексту обращения и базе знаний сформировать короткий ответ, сохранить его как первое сообщение от менеджера и отправить студенту (или только подставлять в тикет для ручной отправки). Ниже в п. 6 — вариант реализации.

---

## 4. Есть ли помощники с ответами в режиме AI?

**Да, частично.** В интерфейсе тикета у менеджера/админа есть:

- **«AI предложить ответ»** — вызов LLM (ключ `chatbot`), в контекст передаётся тема и переписка; модель предлагает краткий ответ, который подставляется в поле ответа; отправить можно только вручную.
- **Шаблоны быстрых ответов** — фиксированные тексты (доступ к курсу, «Мои курсы», «уточняем информацию» и т.д.), без AI.

Отдельного «помощника» с выбором типа вопроса или пошаговыми подсказками нет; по сути помощник — это кнопка «AI предложить ответ» + при необходимости база знаний в контексте (см. п. 5).

## 5. Собирается ли база знаний по ответам и автодответ по типовым вопросам (RAG)?

**Было:**

- **База знаний** есть в «Настройках AI» — один общий текст для чат-бота курса (редактируется вручную). В подсказке ответа по тикету (**suggest-reply**) она **не использовалась** — только тема и переписка.
- Ответы по тикетам **не сохраняются** в базу знаний и не используются для автоматических ответов.

**Сделано:**

- **suggest-reply** доработан: в контекст LLM подмешивается содержимое базы знаний (Настройки AI → база знаний). Модель может опираться на типовые формулировки и правила школы при генерации предложенного ответа (RAG-style без векторного поиска: весь текст БЗ в системном промпте или в блоке контекста).
- Отдельного сбора «базы ответов» из закрытых тикетов пока нет (см. п. 6).

**Возможное развитие (RAG и автодответ):**

- **Вариант A.** Хранить в БД выдержки из базы знаний (или готовые пары «вопрос — ответ») и при suggest-reply / автоответе выбирать релевантные фрагменты по ключевым словам или эмбеддингам (RAG поверх этой коллекции).
- **Вариант B.** Периодически или при закрытии тикета сохранять «разрешённые» пары (тема + первый вопрос + ответ менеджера) в отдельную сущность (например, `SupportAnswer` или раздел в той же БЗ); при новом тикете или при «AI предложить ответ» подтягивать похожие пары в контекст (семантический поиск по эмбеддингам или по ключевым словам).
- **Вариант C.** Один общий текст базы знаний (как сейчас) + ручное дополнение раздела «Типовые ответы поддержки» в Настройках AI; suggest-reply и автоответ продолжают использовать этот текст без отдельного векторного хранилища.

Текущая реализация — шаг к варианту C с возможностью позже добавить A или B.

## 6. План доработок (кратко)

| Задача | Статус / примечание |
|--------|----------------------|
| Ссылка «Тикеты» в сайдбаре админки | Сделано: ведёт на `/portal/manager/tickets`. |
| Подсказка ответа (suggest-reply) с учётом базы знаний | Сделано: в промпт добавляется база знаний из Настроек AI. |
| Автоответ при создании тикета (опционально) | **Сделано.** Настройка в Настройки AI → «Автоответ при создании обращения». Флаг `ticket_auto_reply_enabled` в SystemSetting. При создании тикета с непустым сообщением вызывается LLM (тема + сообщение + БЗ); если ответ «уверенный» (нет фраз типа «не уверен», «обратитесь к менеджеру» и т.д.), он сохраняется как первое сообщение от менеджера и отправляется студенту на email. API: lib/ticket-auto-reply.ts, GET/PATCH /api/portal/admin/ai-settings/ticket-auto-reply. |
| Сбор «базы ответов» из закрытых тикетов | **Сделано.** На странице тикета (только для админа) при статусе «Решён»/«Закрыт» отображается блок «Добавить в базу типовых ответов»: тема + первый вопрос + последний ответ менеджера подставляются в текст, админ может отредактировать и отправить. API `POST /api/portal/admin/ai-settings/knowledge-base/append` дополняет базу знаний. |
| RAG с эмбеддингами | Не реализовано. При росте объёма типовых ответов можно ввести векторное хранилище (например, pgvector при PostgreSQL), индексировать фрагменты БЗ и/или сохранённые ответы, в suggest-reply и автоответе запрашивать топ-N релевантных чанков и подставлять в контекст LLM. |
