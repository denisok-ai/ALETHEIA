# План: иерархические группы (Курсы, Медиатека, Пользователи)

Реализация системы групп по принципам профессиональных LMS: неограниченная вложенность, статические/динамические группы, наследование прав модератора.

---

## 1. Модель данных

- **Group**: id, name, description, parentId (самоссылка), moduleType (course | media | user), type (static | dynamic), accessType (common | personal), displayOrder. Для медиа: smallIcon, largeIcon, showSubgroupsMode; sourceCourseId — если группа медиа создана автоматически по курсу.
- **CourseGroup**, **MediaGroup**, **UserGroup** — связи многие-ко-многим. В UserGroup добавлено поле **role** (member | moderator) для наследования прав на дочерние группы.

---

## 2. API

| Маршрут | Назначение |
|---------|------------|
| GET/POST `/api/portal/admin/groups?moduleType=...` | Список групп, создание |
| GET `/api/portal/admin/groups/tree?moduleType=...` | Дерево для сайдбара |
| GET/PATCH/DELETE `/api/portal/admin/groups/[id]` | Просмотр, изменение, удаление (body: deleteNestedItems — удалить дочерние группы и связи или только отвязать) |
| GET/POST/DELETE `/api/portal/admin/groups/[id]/courses` | Курсы в группе, добавить, убрать |
| GET/POST/DELETE `/api/portal/admin/groups/[id]/media` | Ресурсы в группе |
| GET/POST/DELETE `/api/portal/admin/groups/[id]/users` | Пользователи в группе (с role) |
| GET/POST/DELETE `/api/portal/admin/courses/[courseId]/groups` | Группы курса, добавить в группу, убрать |

---

## 3. UI

- **GroupTree** (components/portal/GroupTree.tsx): дерево групп в сайдбаре, раскрытие/сворачивание, выбор группы; кнопка «Добавить группу» (корень и у каждой группы — подгруппа); при наведении на группу — кнопки «Редактировать» и «Удалить» (onEditGroup, onDeleteGroup с подтверждением на странице).
- **GroupFormModal**: создание/редактирование группы (название, родитель, описание, тип static/dynamic).
- **Вкладка «Группы»** в карточке курса: список групп курса, добавление/удаление привязки.
- **Страница курсов**: слева сайдбар с деревом групп курсов; при выборе группы список курсов фильтруется.

---

## 4. Логика

- **Наследование прав**: модератор родительской группы считается модератором всех дочерних (lib/groups.ts: isModeratorOfGroupOrAncestor).
- **Авто-группа медиа**: при создании курса (POST /api/portal/admin/courses) создаётся группа с moduleType=media, name=title курса, type=dynamic, sourceCourseId=id курса.
- **Удаление группы**: при DELETE с deleteNestedItems=true рекурсивно удаляются дочерние группы и все связи; иначе у дочерних групп сбрасывается parentId, связи с курсами/медиа/пользователями удаляются, сама группа удаляется.

---

## 5. Интеграция по модулям (реализовано)

- **Курсы**: сайдбар с деревом групп, вкладка «Группы» в карточке курса. **Массовые действия:** выбор одного или нескольких курсов (чекбоксы), кнопки «Добавить в группу» (модалка выбора группы) и «Исключить из группы» (при выбранной группе в сайдбаре). Один курс может состоять в нескольких группах. API: `/api/portal/admin/courses/[courseId]/groups`.
- **Медиатека**: сайдбар с деревом групп (moduleType=media), в диалоге редактирования ресурса — блок «Группы ресурса». **Массовые действия:** чекбоксы в таблице, «Добавить в группу» / «Исключить из группы». API: GET/POST/DELETE `/api/portal/admin/media/[id]/groups`.
- **Пользователи**: сайдбар с деревом групп (moduleType=user), вкладка «Группы» в карточке пользователя (роль member/moderator). **Массовые действия:** выбор строк в таблице, «Добавить в группу» (выбор группы в модалке) и «Исключить из группы». API: GET/POST/DELETE `/api/portal/admin/users/[id]/groups`.
- **Страница группы** `/portal/admin/groups/[id]`: просмотр названия, описания, родительской группы, типа; кнопки «Редактировать» (GroupFormModal) и «Удалить группу» (подтверждение, дочерние отвязываются); блок **«Дочерние группы»** (если есть) — ссылки на страницы подгрупп; блок «Состав» — курсы / ресурсы медиатеки / участники в зависимости от moduleType — таблица с кнопкой «Убрать» по строке; кнопка «Добавить» открывает модалку выбора элементов (все курсы, все медиа, все пользователи), добавление выбранных в группу. В дереве групп при наведении — ссылка «Открыть страницу группы» (иконка ExternalLink). GET `/api/portal/admin/media` — список медиа (id, title, type) для пикеров.
