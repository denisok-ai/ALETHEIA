# План доработок модуля «Медиатека»

Документ составлен на основе мини-ТЗ (Mirapolis, макет) и анализа текущей реализации в проекте.

---

## 1. Назначение модуля

Модуль предназначен для **централизованного хранения, описания и управления мультимедийными ресурсами**, используемыми в учебном процессе.

---

## 2. Текущее состояние vs требования

| Требование | Сейчас | Нужно |
|------------|--------|--------|
| **Ресурс: файл или ссылка** | Только файл (upload) | Файл **или** внешняя ссылка (URL) |
| **Категория/тег** | Поле `category` (строка) | Оставить; рекомендованные значения: video, pdf, image |
| **Название** | Есть, обязательно | Без изменений |
| **Описание** | Нет | Добавить поле `description` |
| **Тип (MIME)** | Определяется при загрузке | Без изменений; для ссылок — опционально вручную или по расширению |
| **Предпросмотр** | Только изображения в модалке | Встроенный плеер: изображения, видео (mp4 и др.), PDF, аудио (mp3) |
| **Скачивание** | Всегда по ссылке | Контроль «Разрешить/Запретить скачивание»; при запрете — только просмотр в плеере |
| **Статистика** | Нет | Счётчик просмотров + рейтинг (средняя оценка 1–5) |
| **Поиск** | По названию в списке | Без изменений (уже есть) |
| **Авто-название из имени файла** | Есть (если не задано) | Без изменений |
| **Фильтры** | Тип (все/изображения/PDF/видео), категория | Расширить: тип «Файл» / «Ссылка» + категория + поиск |

## 3. Модель данных (Prisma)

### 3.1. Изменения модели `Media`

| Поле | Сейчас | План |
|------|--------|------|
| `id` | cuid | Без изменений |
| `title` | String | Без изменений |
| `fileUrl` | String | Переименовать в `url` или оставить: для файла — путь `/uploads/media/...`, для ссылки — внешний URL |
| `mimeType` | String? | Без изменений |
| `category` | String? | Без изменений (рекомендуемые: video, pdf, image) |
| `description` | — | **Добавить** `String?` (текст описания ресурса) |
| `type` | — | **Добавить** `String` (enum: `file` \| `link`), по умолчанию `file` |
| `viewsCount` | — | **Добавить** `Int @default(0)` |
| `allowDownload` | — | **Добавить** `Boolean @default(true)` — разрешать ли скачивание |
| `ownerId`, `courseId`, `sortOrder`, `createdAt`, `updatedAt` | Есть | Без изменений |

### 3.2. Новая сущность для рейтинга

- **Вариант A:** поле в `Media`: `ratingSum` (Int), `ratingCount` (Int) — средняя = sum/count.
- **Вариант B:** отдельная таблица `MediaRating` (userId, mediaId, value 1–5), средняя считается запросом.

Рекомендация: **Вариант A** для MVP (проще, быстрее), при необходимости детализации по пользователям — вынести в отдельную таблицу позже.

Итоговая схема (дополнения):

```prisma
model Media {
  id            String   @id @default(cuid())
  title         String
  fileUrl       String   // для type=link храним внешний URL
  mimeType      String?
  category      String?
  description   String?  // новое
  type          String   @default("file")  // "file" | "link"
  viewsCount    Int      @default(0)       // новое
  allowDownload Boolean  @default(true)    // новое
  ratingSum     Int      @default(0)       // новое (сумма оценок)
  ratingCount   Int      @default(0)       // новое (кол-во оценок)
  ownerId       String?
  courseId      String?
  course        Course?  @relation(...)
  sortOrder     Int      @default(0)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

## 4. Форматы и встроенный плеер

### 4.1. Форматы с просмотром в браузере (без скачивания)

| Тип | Расширения / MIME | Примечание |
|-----|-------------------|------------|
| Видео | mpg, avi, mov, flv, mp4, vob, wmv | MP4: желательно H264 + AAC |
| Изображения | png, jpeg, gif, jpg | — |
| Документы | pdf, doc, docx, xls, xlsx, rtf, ppt, pptx, htm, html, txt, odt, odg, odp, odf | — |
| Звук | mp3 | — |

### 4.2. Остальные форматы

- Загрузка **любых** форматов разрешена.
- Если формат не поддерживается плеером — только **скачивание** (если `allowDownload = true`).

### 4.3. Ограничения (по ТЗ)

- Ограничение по **весу** не задано, но есть **таймаут загрузки** (например, 30 мин).
- При большом размере файла (например, > 100 МБ) конвертация/предпросмотр может быть недоступна — показывать предупреждение или только ссылку на скачивание.

## 5. Функционал по экранам

### 5.1. Админка: загрузка и добавление ресурсов

- **Загрузка файлов:** выбор в проводнике + drag-and-drop (уже есть).
- **Расширить типы принимаемых файлов:** не только image/pdf/video, но и doc, xls, ppt, txt, mp3 и др. (по списку из п. 4), при необходимости — «все файлы».
- **Добавление ссылки:** форма «Тип: Ссылка», поле URL, название, категория, описание. Без файла.
- **Название:** если при загрузке файла не задано — подставлять имя файла (уже есть).
- **Поля формы:** Название, Категория (выбор: video / pdf / image или произвольная), Описание (textarea), для ссылки — URL. Кнопка «Загрузить» / «Добавить ссылку».

### 5.2. Админка: список ресурсов

- **Таблица:** №, Название, Категория, Тип (Файл/Ссылка), MIME/тип, Просмотры, Рейтинг (средняя 1–5), Скачивание (да/нет), Действия.
- **Фильтры:** «Тип» (Все / Файл / Ссылка), «Категория» (выпадающий список), поиск «Найти в списке» по названию (уже есть).
- **Действия над ресурсом:**
  - **Предпросмотр** — модальное окно с встроенным плеером: изображение (img), видео (video), аудио (audio), PDF (iframe или object). Для остальных — сообщение «Доступно только скачивание» и кнопка «Скачать».
  - **Скачать** — ссылка на файл/URL с атрибутом download или открытие в новой вкладке (в зависимости от `allowDownload`).
  - **Редактировать** — метаданные: название, категория, описание, разрешить/запретить скачивание.
  - **Удалить** — с подтверждением (уже есть).

### 5.3. Контроль доступа к скачиванию

- В карточке/форме редактирования: чекбокс или переключатель «Разрешить скачивание».
- Если запрещено — в интерфейсе студента показывать только кнопку «Смотреть/Слушать» (плеер), без кнопки «Скачать».

### 5.4. Статистика и рейтинг

- **Просмотры:** при открытии ресурса студентом (просмотр в плеере или переход по ссылке) увеличивать `viewsCount` (один раз за сессию/за день на ресурс и пользователя — по решению).
- **Рейтинг:** в интерфейсе студента — блок «Оценить» (звёзды 1–5). Отправка оценки на API, пересчёт среднего (ratingSum, ratingCount) и отображение в админке в таблице (колонка «Рейтинг»).

---

## 6. Студенческий интерфейс (Медиатека)

- Список ресурсов: карточки или таблица с названием, категорией, типом (файл/ссылка).
- При клике: открытие в плеере (если формат поддерживается) или скачивание (если разрешено).
- Для ресурсов с плеером: одна кнопка «Смотреть/Слушать»; при `allowDownload = true` — вторая «Скачать».
- Блок оценки (звёзды 1–5) и отображение среднего рейтинга и/или количества просмотров (опционально).

## 7. API

| Метод | Маршрут | Назначение |
|-------|---------|------------|
| GET | `/api/portal/admin/media` | Список всех ресурсов (для таблицы с пагинацией при необходимости) |
| POST | `/api/portal/admin/media/upload` | Загрузка файла (тело multipart) — без изменений по смыслу, добавить поля description, allowDownload |
| POST | `/api/portal/admin/media/link` или в одном upload с type=link | Добавление ресурса-ссылки (JSON: url, title, category, description) |
| GET | `/api/portal/admin/media/[id]` | Один ресурс (уже есть) |
| PATCH | `/api/portal/admin/media/[id]` | Обновление (title, category, description, allowDownload) — расширить |
| DELETE | `/api/portal/admin/media/[id]` | Удаление (уже есть) |
| GET | `/api/portal/media` или `/api/portal/student/media` | Список ресурсов для студента (для страницы «Медиатека») |
| GET | `/api/portal/media/[id]/view` или встроить в страницу | Открытие ресурса: увеличить viewsCount, вернуть URL или stream для плеера |
| POST | `/api/portal/media/[id]/rate` | Оценка 1–5 от студента (обновить ratingSum, ratingCount) |

## 10. Критерии приёмки (MVP)

- Ресурс может быть **файлом** (загрузка) или **ссылкой** (URL).
- У ресурса есть: название, категория (рекомендуемые video/pdf/image), описание, тип (file/link), разрешение на скачивание.
- В админке: загрузка (в т.ч. drag-and-drop), добавление ссылки, таблица с фильтрами по типу и категории, поиск по названию.
- Предпросмотр в админке: изображения, видео, аудио, PDF во встроенном плеере.
- Редактирование метаданных и переключатель «Разрешить скачивание».
- Удаление с подтверждением.
- Счётчик просмотров и рейтинг (1–5), отображаются в админке.
- Студент: список ресурсов, просмотр в плеере (где возможно), скачивание только если разрешено, возможность поставить оценку.

После выполнения плана модуль «Медиатека» будет соответствовать мини-ТЗ и готов к дальнейшему расширению (например, теги, конвертация, интеграция с Google Docs).
