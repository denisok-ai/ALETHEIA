## 2. Пробелы: функционал без UI или с неполным UI

### 2.1 Критичные

- **Нет** критичных пробелов: основные сценарии (оплата, курсы, пользователи, рассылки, CRM, отчётность, мониторинг, настройки) закрыты интерфейсом.

### 2.2 Желательные доработки

| № | Что | Где сейчас | Предложение |
|---|-----|------------|-------------|
| 1 | **Список отписавшихся от рассылок** | Модель `MailingUnsubscribe`, исключение при отправке рассылки. Нет экрана в админке. | Добавить в раздел «Рассылки» или «Настройки» блок/страницу «Отписавшиеся»: таблица email + дата, экспорт CSV, при необходимости удаление из списка (повторная подписка). |
| 2 | **Централизованный раздел «Группы»** | Группы доступны только из контекста Курсов, Медиатеки, Пользователей (дерево + страница группы). | Опционально: пункт меню «Группы» с тремя подразделами (Курсы / Медиатека / Пользователи) или одна страница с табами — для тех, кто хочет управлять группами без входа в модуль. |
| 3 | **Сброс пароля по email** | В локальном режиме отключён, показ «Обратитесь к администратору». | Для продакшена: при наличии настройки SMTP/Resend — включить отправку ссылки на сброс и страницу `auth/update-password`. |

### 2.3 Мелкие улучшения

| № | Что | Предложение |
|---|-----|-------------|
| 4 | **Единая точка входа «Помощь»** | В портале (студент/менеджер/админ) вынести в шапку или футер ссылку на раздел поддержки или статичную страницу «Помощь» (часто задаваемые вопросы, контакты). |
| 5 | **Подтверждение при несохранённых данных** | При уходе со страницы редактирования (курс, рассылка, шаблон и т.д.) с изменёнными полями — предупреждение «Есть несохранённые изменения». |
| 6 | **Клавиатурная навигация и фокус** | Проверить модальные окна (фокус в ловушке, закрытие по Escape), кнопки «Назад» и порядок табуляции в формах. |

## 3. Дизайн и консистентность

### 3.1 Что проверено

- **Цвета:** в `globals.css` и `tailwind.config.ts` заданы primary (#2D1B4E), secondary (#D4AF37), dark (#0A0E27), bg, text-muted — соответствуют Spec и Project.
- **Шрифты:** Literata (заголовки), Outfit (текст) подключены в `app/layout.tsx`.
- **Портал:** единые компоненты PageHeader, Breadcrumbs, Card; таблицы с общими паттернами (фильтры, пагинация, EmptyState, скелетоны).
- **Главная:** секции Hero, About, NewsWidget, Program, Author, Testimonials, Pricing, FAQ, Contact; Header и Footer (Footer только на публичных страницах).

### 3.2 Рекомендации по дизайну

| № | Область | Рекомендация |
|---|---------|--------------|
| 1 | **Состояния загрузки** | Убедиться, что у всех тяжёлых списков и форм есть скелетон или спиннер (уже есть во многих разделах; проверить новые страницы). |
| 2 | **Сообщения об ошибках** | В формах и при действиях везде показывать понятные сообщения (toast или inline), не только в консоли. |
| 3 | **Адаптив** | Критические экраны (оплата, логин, основные таблицы админки) проверить на мобильных и планшетах. |
| 4 | **Доступность** | Кнопки без текста (только иконка) должны иметь `aria-label`; у ключевых действий — видимый фокус. |

## 4. План доработок (приоритеты)

### Высокий приоритет

1. **Список отписавшихся (MailingUnsubscribe)** — **выполнено (2025-03-10)**  
   - API: `GET /api/portal/admin/mailings/unsubscribed` (JSON список; `?format=csv` — экспорт), `DELETE /api/portal/admin/mailings/unsubscribed/[id]` (удалить из списка).  
   - UI: на странице «Рассылки» блок «Отписавшиеся от рассылок»: таблица (№, email, дата), кнопка «Экспорт CSV», удаление строки с подтверждением.

### Средний приоритет

2. **Сброс пароля по email в продакшене**  
   - Проверить наличие письма со ссылкой на сброс и страницу `auth/update-password` при включённой почте. Текущее состояние: страницы `/reset-password` и `/auth/update-password` — заглушки («Обратитесь к администратору»). Для полноценного сброса по email нужны: API запроса сброса (отправка письма с токеном), модель/хранилище токена, API установки нового пароля по токену.

3. **Предупреждение о несохранённых изменениях** — **выполнено (2025-03-10)**  
   - Хук `lib/useUnsavedChanges(isDirty)`: при закрытии вкладки/обновлении страницы показывается стандартный диалог браузера.  
   - Подключено в формах: **Шаблон уведомления** (создание/редактирование), **Шаблон сертификата** (создание/редактирование). При нажатии «К списку» / «Отмена» при наличии несохранённых изменений показывается confirm «Есть несохранённые изменения. Уйти без сохранения?».

### Низкий приоритет

4. **Раздел «Группы» в меню админки** — **выполнено (2025-03-10)**  
   - Пункт «Группы» в блоке «Контент и обучение»: страница-хаб `/portal/admin/groups` с тремя карточками (Группы курсов, Группы медиатеки, Группы пользователей) и ссылками в разделы Курсы, Медиатека, Пользователи.

5. **Страница «Помощь» в портале** — **выполнено (2025-03-10)**  
   - Страницы «Помощь» для студента (`/portal/student/help`), админа (`/portal/admin/help`) и менеджера (`/portal/manager/help`). Общий контент: FAQ (курсы, пароль, поддержка), ссылка на раздел поддержки/тикетов. Пункт «Помощь» добавлен в сайдбар каждой роли.

6. **Аудит доступности** — **выполнено (2025-03-10)**  
   - **Dialog (components/ui/dialog.tsx):** при открытии фокус переходит на первый фокусный элемент внутри модалки; Tab/Shift+Tab замыкаются внутри (focus trap); при закрытии фокус возвращается на ранее активный элемент; Escape по-прежнему закрывает модалку.  
   - **aria-label:** у кнопок только с иконкой (пагинация, действия в таблицах, сайдбар, GroupTree, чат и т.д.) уже добавлены или добавлены недостающие (пагинация в PaymentsTableClient — «Предыдущая страница» / «Следующая страница»).

## 5. Итог

- **Функционал:** Все основные API имеют соответствующий интерфейс (админка, студент, менеджер, публичные страницы). Список отписавшихся от рассылок выведен в админке (блок на странице «Рассылки»).
- **Дизайн:** Палитра и типографика соответствуют документации; в портале используются общие компоненты и паттерны.
- **План:** В первую очередь — вывод списка отписавшихся в админке; затем — сброс пароля по email и предупреждение о несохранённых данных; по возможности — раздел «Группы», страница «Помощь» и доработки доступности.
